Implementation flow

Let reasoning arrive while it is written.

Reuse the proven assistant-stream transport, but preserve reasoning’s distinct timeline type, request tracking label, and collapsed final state.

1

Create a transient entry

Add an empty, typed reasoning message to the user projection before the network request begins.

2

Stream cumulative text

Call the existing SSE endpoint and replace the transient entry’s content on every received chunk.

3

Persist atomically

Save the completed reasoning event, then remove the transient entry. On failure, remove it without saving.

4

Shape the UI state

Keep live reasoning expanded so text is visible, then let the persisted message return to the normal collapsed view.

Decisions

  • Reasoning remains a separate chat event and message type.
  • The stream is transient and never enters LLM context.
  • Request inspection continues to identify the call as “Reasoning.”

Completion criteria

  • Tokens visibly update before generation completes.
  • No empty or partial reasoning is persisted after failure.
  • Assistant streaming and reasoning-disabled flows remain unchanged.
  • Projection, service, transport, and rendering tests pass.
