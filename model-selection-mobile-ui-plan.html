Mobile interaction plan

01

Open to remembered setups

Current, saved, and recent configurations are the first screen. The raw model catalog remains one tab away.

02

Navigate instead of expanding

A tap opens a dedicated configuration screen. Advanced fields never unfold inside a scrolling result row.

03

Keep the decision reachable

Save and apply actions stay above the safe-area inset, while Back preserves the draft for the current session.

04

Scale up progressively

Use this same list-to-detail structure on tablets. Only wide desktop screens graduate to simultaneous two-pane display.

Core responsive decision

The mobile flow is the canonical interaction model. Desktop is an enhancement that places the same two screens side by side—not a separate experience that mobile must imitate.
